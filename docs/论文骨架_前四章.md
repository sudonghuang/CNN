# 摘 要

**中文摘要**（约300字）

随着高校教学管理信息化水平的不断提升，传统人工点名考勤方式因效率低下、易于作弊等问题已难以满足现代化教学管理需求。本文设计并实现了一套基于卷积神经网络（CNN）的校园课堂智能考勤系统。

系统采用前后端分离架构，后端基于Flask框架提供RESTful API及WebSocket实时推送服务，前端基于Vue 3框架实现响应式Web界面。AI识别模块采用MTCNN进行多尺度人脸检测，分别以VGG16和ResNet50为骨干网络进行两阶段迁移学习，提取512维L2归一化特征向量，通过余弦相似度匹配实现身份识别。系统设计了双阈值判断机制，将识别结果分为"出勤"、"待核实"和"缺勤"三种状态，提升了实际应用中的鲁棒性。

在LFW数据集Top-50人物子集上的实验结果表明，ResNet50模型验证集Top-1准确率达到86.30%，AUC-ROC达到99.30%，误识率（FAR）仅为1.17%，单帧推理延迟176.5ms，满足预期性能目标。通过与VGG16的架构对比分析，ResNet50在参数量（25M vs 138M）、计算效率和部署可行性方面均具有显著优势，更适合资源受限环境下的实时推理部署。系统实现了学生信息管理、人脸数据采集、实时考勤识别、考勤记录查询及报表统计等核心功能，支持管理员、教师、辅导员、学生四种角色的差异化权限管理。

**Abstract**（约200字英文）

With the continuous advancement of information technology in university teaching management, traditional manual attendance methods can no longer meet modern demands due to low efficiency and susceptibility to fraud. This paper designs and implements an intelligent classroom attendance system based on Convolutional Neural Networks (CNN).

The system adopts a front-end and back-end separated architecture. MTCNN is employed for multi-scale face detection, while ResNet50 serves as the backbone network for two-phase transfer learning, extracting 512-dimensional L2-normalized feature vectors for identity recognition via cosine similarity matching. Experiments on the LFW Top-50 subset demonstrate that the ResNet50 model achieves a validation Top-1 accuracy of 86.30%, AUC-ROC of 99.30%, and a false acceptance rate (FAR) of only 1.17%, with an inference latency of 176.5ms per frame on CPU. Architectural comparison with VGG16 (138M parameters) confirms ResNet50's significant advantage in parameter efficiency (25M parameters) and deployment feasibility under resource-constrained environments.

**Keywords**: Convolutional Neural Network; Face Recognition; Transfer Learning; Attendance System; ResNet50; VGG16

## 1.1 研究背景与意义

### 1.1.1 传统考勤方式的局限性

## 2.1 卷积神经网络基础

### 2.1.1 CNN的基本结构

卷积神经网络（Convolutional Neural Network, CNN）是深度学习在计算机视觉领域的核心模型。标准CNN由卷积层、池化层和全连接层组成【可配图：典型CNN结构示意图】。

**卷积层**通过卷积核在输入特征图上滑动，提取局部空间特征，具有局部连接和权值共享的特性，大幅减少了参数量。给定输入特征图 $X$，卷积操作可表示为：

$$Y_{i,j} = \sum_{m}\sum_{n} X_{i+m, j+n} \cdot W_{m,n} + b$$

**批归一化（Batch Normalization）**【引用 Ioffe and Szegedy, 2015】加速训练收敛并缓解内部协变量偏移，已成为现代CNN的标准组件。

### 2.1.2 VGG16网络结构

VGGNet【引用 Simonyan and Zisserman, 2014】由牛津大学视觉几何组提出，以其简洁一致的3×3卷积设计著称。VGG16包含13个卷积层和3个全连接层，共约1.38亿参数【可配图：VGG16网络结构】。

VGG16采用多个3×3卷积堆叠来等效更大感受野（两个3×3等效一个5×5），在减少参数的同时引入更多非线性，特征提取能力强，但参数量大、计算成本高。

本文在VGG16基础上，将原始全连接分类头替换为：
$$\text{FC}(25088 \to 4096) \to \text{ReLU} \to \text{Dropout} \to \text{FC}(4096 \to 512) \to \text{ReLU} \to \text{Dropout} \to \text{FC}(512 \to C)$$
其中倒数第二层的512维输出即作为人脸特征向量。

### 2.1.3 ResNet50网络结构与残差连接

ResNet【引用 He et al., 2015】提出残差学习（Residual Learning）解决深层网络退化问题。其核心创新是跳跃连接（Skip Connection）：

$$\mathbf{y} = \mathcal{F}(\mathbf{x}, \{W_i\}) + \mathbf{x}$$

网络只需学习残差映射 $\mathcal{F}(\mathbf{x})$，而非完整映射，极大地缓解了梯度消失问题【可配图：残差块结构】。

ResNet50采用Bottleneck结构（1×1→3×3→1×1），深度50层，参数约2500万，远少于VGG16的1.38亿，在精度与效率上取得更好平衡。本文将ResNet50的全连接头替换为：
$$\text{FC}(2048 \to 512) \to \text{ReLU} \to \text{Dropout} \to \text{FC}(512 \to C)$$

## 2.2 迁移学习

### 2.2.1 迁移学习原理

迁移学习（Transfer Learning）是将在源域（Source Domain）上习得的知识迁移到目标域（Target Domain）的机器学习方法【引用 Pan and Yang, 2010】。在人脸识别领域，ImageNet预训练模型已学习到丰富的纹理、边缘、形状等底层特征，这些特征对人脸识别同样有效。

相较于从头训练，迁移学习的优势在于：（1）所需标注数据量大幅减少；（2）训练收敛更快；（3）在小样本场景下具有更强的泛化能力。

### 2.2.2 两阶段微调策略

本系统采用两阶段迁移学习策略【可配图：两阶段训练流程图】：

**阶段一：冻结主干，训练分类头（Epochs 1~10）**

冻结预训练骨干网络的所有卷积层参数，仅训练新增分类头。使用较高学习率（$lr = 10^{-3}$），使随机初始化的分类头在ImageNet特征空间上快速收敛，避免反向传播破坏已学习的特征。

**阶段二：解冻全部，端到端微调（Epochs 11~30）**

解冻所有参数，使用SGD+Momentum优化器（$lr = 10^{-4}$）配合余弦退火学习率调度，在保留预训练特征的基础上对全网络进行细粒度调整，使特征提取器适配人脸识别目标域。

采用SGD而非Adam用于微调阶段，原因在于SGD在CPU环境下内存占用约为Adam的1/3，且有研究表明SGD+Momentum在微调任务上泛化性能更优【引用】。

## 2.3 人脸检测：MTCNN

### 2.3.1 MTCNN基本原理

MTCNN（Multi-task Cascaded Convolutional Networks）【引用 Zhang et al., 2016】是一种基于级联CNN的人脸检测与关键点定位方法，由三个子网络串联构成【可配图：MTCNN三阶段流程】：

**P-Net（Proposal Network）**：快速扫描全图，生成候选人脸框（Bounding Box）和面部关键点初始估计，大量过滤背景区域；

**R-Net（Refine Network）**：对P-Net输出的候选框进行精细化筛选和回归，进一步提高定位精度；

**O-Net（Output Network）**：最终输出精确的人脸框和5个面部关键点（双眼、鼻尖、双嘴角）坐标。

### 2.3.2 相比传统方法的优势

传统Haar Cascade人脸检测对光照和角度敏感，误检率高；MTCNN通过多任务联合学习（分类+回归+关键点），在多尺度、多角度、复杂光照条件下均表现出更强的鲁棒性，检测精度较Haar Cascade提升约15%【引用】。

## 2.4 特征匹配：余弦相似度与双阈值机制

### 2.4.1 L2归一化与余弦相似度

提取的特征向量经L2归一化后，欧氏距离与余弦相似度等价。对于归一化向量 $\hat{f}_i = f_i / \|f_i\|_2$，两向量的余弦相似度为：

$$\text{sim}(\hat{f}_q, \hat{f}_k) = \hat{f}_q \cdot \hat{f}_k = \sum_{d=1}^{512} \hat{f}_q^{(d)} \cdot \hat{f}_k^{(d)}$$

取值范围为[-1, 1]，值越大表示越相似。归一化后的批量匹配可用矩阵点积高效计算：$\mathbf{S} = \hat{F}_{\text{query}} \cdot \hat{F}_{\text{db}}^T$，时间复杂度为 $O(N \cdot d)$，百人规模下匹配耗时不足1ms。

### 2.4.2 双阈值判断机制

单一阈值在实际应用中存在拒识率和误识率难以同时优化的矛盾。本系统设计双阈值机制：

| 相似度 $s$ | 判断结果 | 含义 |
|---|---|---|
| $s \geq 0.75$ | 出勤（present） | 高置信度识别成功 |
| $0.50 \leq s < 0.75$ | 待核实（unverified） | 识别结果存疑，待人工复核 |
| $s < 0.50$ | 未知（rejected） | 非注册学生，忽略 |

"待核实"状态引入了人工监督环节，在保证系统自动化的同时降低了误识风险，更贴近实际教学管理需求。

## 2.5 Web技术栈

### 2.5.1 Flask框架

Flask【引用】是基于Python的轻量级WSGI Web框架，以其灵活的蓝图（Blueprint）机制和丰富的扩展生态（Flask-JWT-Extended、Flask-SQLAlchemy、Flask-SocketIO）著称，适合快速构建RESTful API服务。

### 2.5.2 Socket.IO实时通信

传统HTTP轮询在实时场景下存在延迟高、服务器压力大等问题。Socket.IO基于WebSocket协议实现全双工实时通信，支持房间（Room）机制，可将识别结果精确推送至指定考勤任务的监控界面，显著降低端到端延迟。

### 2.5.3 Vue 3与Pinia

Vue 3【引用】基于Composition API提供更灵活的状态组织方式，配合Pinia状态管理库和Element Plus UI组件库，实现响应式的单页面应用（SPA）。Pinia的`attendanceStore`集中管理WebSocket连接、摄像头帧抓取和识别状态，将UI与业务逻辑有效解耦。

### 2.5.4 JWT身份认证

JSON Web Token（JWT）【引用】是一种无状态的身份认证方案，服务端无需存储Session，Token由Header.Payload.Signature三部分组成，Payload中携带用户ID和角色信息，便于实现无状态的分布式权限控制。

### 3.3.6 报表统计模块（F6）

## 4.3 AI模块设计

### 4.3.1 模块整体架构

AI模块由五个子模块组成，通过`ModelManager`单例统一对外提供服务【可配图：AI模块类图】：

| 子模块 | 职责 |
|--------|------|
| `FaceDetector` | 封装MTCNN，输入帧图像，输出人脸Bounding Box列表 |
| `FacePreprocessor` | 按框裁剪人脸区域，缩放至224×224，ImageNet归一化 |
| `FaceRecognizer` | 加载训练好的VGG16/ResNet50权重，前向传播到512维特征层 |
| `FeatureDatabase` | 加载特征库numpy矩阵，执行L2归一化和批量余弦相似度匹配 |
| `ModelManager` | 单例，协调上述四个模块，提供`recognize_faces(frame)`接口 |

### 4.3.2 推理流程设计

```
输入: BGR图像帧（OpenCV格式）
    │
    ▼ FaceDetector.detect()
[MTCNN检测] → 返回[(x1,y1,x2,y2), ...]
    │
    ▼ FacePreprocessor.process()
[裁剪+缩放+归一化] → 返回 Tensor [N, 3, 224, 224]
    │
    ▼ FaceRecognizer.extract()
[ResNet50前向传播，截断到fc[:2]] → 512维向量 [N, 512]
    │
    ▼ L2归一化
[向量 / ||向量||₂] → 单位向量 [N, 512]
    │
    ▼ FeatureDatabase.match()
[矩阵点积] S = Query × DB^T → 相似度矩阵 [N, M]
    │
    ▼ 双阈值判断
[s≥0.75 → present; 0.5≤s<0.75 → unverified; s<0.5 → reject]
    │
    ▼
输出: [{student_id, student_db_id, status, confidence}, ...]
```

### 4.3.3 训练模块设计

训练脚本`model/train.py`实现两阶段迁移学习【可配图：训练流程图】：

**数据集组织**：使用`torchvision.datasets.ImageFolder`读取`dataset/faces/train/`目录，文件夹名即类别标签，无需额外标注文件；验证集从`dataset/faces/val/`读取。

**数据增强**（训练集）：随机裁剪（256→224）、随机水平翻转、颜色抖动（亮度/对比度/饱和度各±30%）、随机旋转（±15°）、ImageNet均值标准差归一化。

**两阶段策略**：
- Phase 1（前1/3轮）：冻结骨干，Adam优化器 $lr=10^{-3}$，固定学习率，分类头快速收敛；
- Phase 2（后2/3轮）：解冻全部，SGD+Momentum优化器 $lr=10^{-4}$，余弦退火调度。

**最优模型保存**：以验证集准确率为依据，每当val_acc创历史最高时保存权重文件，同时记录num_classes、best_acc、best_epoch至JSON元信息文件，供推理时自动加载。

### 4.4.3 JWT权限控制实现

```python
# 核心装饰器设计
def require_roles(*roles):
    def decorator(f):
        @wraps(f)
        def wrapper(*args, **kwargs):
            token = request.headers.get('Authorization', '').replace('Bearer ', '')
            try:
                payload = jwt.decode(token, SECRET_KEY, algorithms=['HS256'])
            except jwt.ExpiredSignatureError:
                return error("Token已过期", 401)
            if payload['role'] not in roles:
                return error("权限不足", 403)
            g.current_user_id = payload['sub']
            g.current_role    = payload['role']
            return f(*args, **kwargs)
        return wrapper
    return decorator
```
