## 1. 系统整体业务流程

```mermaid
flowchart TD
    Start([系统启动]) --> Login[用户登录]
    Login --> AuthCheck{身份验证}
    AuthCheck -->|失败| LoginFail[显示错误提示] --> Login
    AuthCheck -->|成功| RoleRoute{角色路由}

    RoleRoute -->|管理员| AdminDash[管理员控制台]
    RoleRoute -->|教师| TeacherDash[教师控制台]
    RoleRoute -->|辅导员| CounselorDash[辅导员控制台]
    RoleRoute -->|学生| StudentDash[学生查询页面]

    AdminDash --> AdminOps[学生管理 / 人脸录入 / 系统设置]
    TeacherDash --> TeacherOps[发起考勤 / 查看记录 / 导出报表]
    CounselorDash --> CounselorOps[全院查询 / 预警管理 / 报表导出]
    StudentDash --> StudentOps[查询个人考勤记录]

    AdminOps --> FaceRegist[人脸注册流程]
    TeacherOps --> AttendProc[考勤任务流程]

    FaceRegist --> ModelTrain{触发训练?}
    ModelTrain -->|是| TrainModel[模型增量训练]
    ModelTrain -->|否| Done1([完成])
    TrainModel --> Done1

    AttendProc --> ReportGen[考勤报表生成]
    ReportGen --> Done2([完成])
```

## 2. 用户登录与认证流程

```mermaid
flowchart TD
    A([开始]) --> B[用户输入账号/密码]
    B --> C[POST /api/auth/login]
    C --> D{账号是否存在?}
    D -->|否| E[返回 401: 账号不存在] --> B
    D -->|是| F{账号是否启用?}
    F -->|否| G[返回 403: 账号已禁用] --> End
    F -->|是| H{bcrypt 密码验证}
    H -->|失败| I[返回 401: 密码错误] --> B
    H -->|成功| J[生成 JWT Token\nexpires=24h]
    J --> K[返回 Token + 用户角色信息]
    K --> L[前端存储 Token\n至 localStorage/Pinia]
    L --> M[路由跳转至对应控制台]
    M --> End([登录完成])
```

## 3. 学生人脸注册流程

```mermaid
flowchart TD
    A([管理员发起注册]) --> B[选择目标学生]
    B --> C{选择采集方式}

    C -->|上传图片| D["上传人脸图片 >=5张"]
    C -->|实时采集| E[开启摄像头\n实时预览]

    D --> F[POST /api/faces/upload]
    E --> G[每次点击拍照\nWebcam 截帧]
    G --> H{"已采集图片>=5张?"}
    H -->|否| G
    H -->|是| F

    F --> I[OpenCV MTCNN\n人脸检测]
    I --> J{检测到人脸?}
    J -->|否| K[返回错误: 未检测到人脸\n提示重新上传] --> C
    J -->|是| L["裁剪人脸区域\n缩放 224x224\n归一化"]
    L --> M["保存至\ndataset/faces/student_id/"]
    M --> N[更新 face_images 表\nface_count += 1]
    N --> O{是否触发训练?}
    O -->|否| P([注册完成])
    O -->|是| Q[POST /api/faces/train]
    Q --> R[加载数据集\nDataLoader]
    R --> S[VGG16 / ResNet50\n迁移学习微调]
    S --> T["保存新模型权重\ncheckpoints/"]
    T --> U[更新 model_manager\n热加载新特征库]
    U --> P
```

## 4. 考勤任务核心流程（实时摄像头）

```mermaid
flowchart TD
    A([教师发起考勤]) --> B[选择课程 + 日期]
    B --> C[POST /api/attendance/tasks\n创建任务记录 status=pending]
    C --> D[POST /api/attendance/tasks/id/start]
    D --> E[后端加载课程学生名单\n加载模型特征库]
    E --> F[建立 WebSocket 连接\n开启摄像头帧推送]

    F --> G[前端持续采集摄像头帧\n每 500ms 推送一帧]
    G --> H[后端: MTCNN 人脸检测]
    H --> I{检测到人脸?}
    I -->|否| J[继续下一帧] --> G
    I -->|是| K[OpenCV 预处理\n裁剪+归一化]
    K --> L[CNN 特征提取\n输出特征向量]
    L --> M[余弦相似度匹配\n与全班特征对比]
    M --> N{最高相似度值}

    N -->|">=0.75"| O[识别成功]
    N -->|"0.5~0.75"| P[标记 unverified]
    N -->|"<0.5"| Q[未知人员\n不写库]

    O --> R{该学生本次已记录?}
    R -->|是| S[跳过\n去重] --> G
    R -->|否| T[写入 attendance_records\nstatus=present\nconfidence=similarity]
    T --> U[WebSocket 推送识别结果至前端\n实时高亮学生姓名]
    U --> G

    P --> V[写入 attendance_records\nstatus=unverified]
    V --> W[前端显示待核实标记] --> G

    G --> X{教师点击结束?}
    X -->|否| G
    X -->|是| Y[POST /api/attendance/tasks/id/stop]
    Y --> Z[未出现学生自动标记 absent]
    Z --> AA[更新任务 status=finished\n统计 present_count]
    AA --> AB[WebSocket 关闭\n摄像头释放]
    AB --> AC([考勤结束\n展示摘要])
```

## 6. CNN 模型推理详细流程

```mermaid
flowchart LR
    A["输入: 图像帧\nHxWx3"] --> B

    subgraph B[MTCNN 人脸检测]
        B1[P-Net\n快速候选框] --> B2[R-Net\n候选框精化]
        B2 --> B3[O-Net\n最终定位+关键点]
    end

    B --> C[输出: BBox列表\n每个人脸坐标]

    C --> D

    subgraph D[OpenCV 预处理]
        D1["按BBox裁剪\n+20px边距"] --> D2["Resize 224x224"]
        D2 --> D3[转 RGB\n/255.0]
        D3 --> D4[减 ImageNet 均值\n除标准差]
    end

    D --> E["Tensor: 1x3x224x224"]

    E --> F

    subgraph F[CNN 骨干网络]
        F1{模型选择}
        F1 -->|VGG16| F2[Conv Blocks 1-5\nMaxPool]
        F1 -->|ResNet50| F3[ResidualBlocks\n4个Stage]
        F2 --> F4[去掉 FC 分类头\n取 AvgPool 输出]
        F3 --> F4
    end

    F --> G[特征向量\n512d / 2048d]
    G --> H[L2 归一化\n单位球面]

    H --> I

    subgraph I[相似度匹配]
        I1["加载内存特征库\nstudent_id -> feature_vec"]
        I2["余弦相似度计算\ncos_sim = dot(a,b) / norm(a)*norm(b)"]
        I1 --> I2
        I2 --> I3[取最高相似度\n及对应 student_id]
    end

    I --> J{阈值判断}
    J -->|">=0.75"| K["识别成功\n返回 student_id + confidence"]
    J -->|"0.5~0.75"| L["待核实\n返回 unverified"]
    J -->|"<0.5"| M["未知\n不写库"]
```

## 7. 异常考勤预警流程

```mermaid
flowchart TD
    A([定时任务: 每日凌晨触发]) --> B[查询各课程近期考勤记录]
    B --> C[按学生分组统计\n最近N次考勤缺勤次数]
    C --> D{"连续缺勤>=3次?"}
    D -->|否| E([本次检查结束])
    D -->|是| F[生成预警记录\n写入 warnings 表]
    F --> G{辅导员是否绑定邮件?}
    G -->|是| H[发送预警邮件通知]
    G -->|否| I[仅在系统内显示预警]
    H --> J[系统预警列表更新]
    I --> J
    J --> E
```
