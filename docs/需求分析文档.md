## 1. 项目背景与目标

### 1.1 背景

高校课堂考勤是教学管理的重要环节，传统考勤方式（点名、签到表、刷卡）存在以下痛点：

- **效率低下**：逐一点名耗费课堂时间（大班级 10～20 分钟）
- **代签风险**：签到表、刷卡卡片易被他人代签，考勤数据失真
- **数据孤岛**：纸质记录难以统计、追溯，报表生成费力
- **管理成本高**：辅导员/教务处核查工作量大

### 1.2 目标

设计并实现一套基于 CNN 人脸识别的智能考勤系统，具体目标如下：

| 目标维度 | 描述 |
|----------|------|
| 功能目标 | 通过摄像头自动识别课堂学生身份，完成无感知考勤 |
| 性能目标 | 人脸检测准确率≥90%，身份识别准确率≥85%，响应时间≤5秒/次 |
| 管理目标 | 支持 Web 端对学生信息、考勤记录的全生命周期管理 |
| 研究目标 | 对比 VGG16 与 ResNet50 在人脸识别任务上的效果差异 |

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 人脸检测准确率 | ≥ 90% | 在正常教室光线下 |
| 身份识别准确率 | ≥ 85% | 注册图片≥5张/人时 |
| 单次考勤响应时间 | ≤ 5秒 | 从摄像头捕帧到写库完成 |
| 前端页面加载时间 | ≤ 3秒 | 首屏 |
| 并发支持 | ≥ 10 路摄像头同时考勤 | 后端异步处理 |

### 4.2 安全需求

| 编号 | 描述 |
|------|------|
| S-01 | 用户密码 bcrypt 加密存储，不可逆 |
| S-02 | 接口统一 JWT 鉴权，Token 有效期 24 小时 |
| S-03 | 人脸图片存储与传输脱敏（不含身份证等敏感信息） |
| S-04 | SQL 参数化查询，防止 SQL 注入 |
| S-05 | 文件上传限制类型和大小（图片≤5MB，视频≤500MB） |

### 4.3 可用性需求

- 系统可用性 ≥ 99%（非维护期间）
- 支持主流浏览器（Chrome、Edge、Firefox 最新版本）
- 前端响应式布局，支持 1080p 以上分辨率

### 4.4 可维护性需求

- 后端遵循 RESTful 接口规范，提供 Swagger 文档
- 代码注释覆盖率 ≥ 30%
- 模型与业务代码解耦，支持替换 CNN 骨干网络

### UC-01：发起人脸考勤

| 项目 | 内容 |
|------|------|
| **用例名称** | 发起人脸考勤 |
| **参与者** | 教师 |
| **前置条件** | 教师已登录；当前课程已有学生人脸数据注册 |
| **主成功场景** | 1. 教师选择课程和上课时间 → 2. 点击"开始考勤" → 3. 系统开启摄像头 → 4. 系统逐帧检测人脸 → 5. 识别匹配学生身份 → 6. 出勤记录写入数据库 → 7. 教师点击"结束考勤" → 8. 系统生成本次考勤摘要 |
| **扩展场景** | 4a. 未检测到人脸：系统提示"未检测到人脸，请调整摄像头角度" |
|  | 5a. 识别置信度<阈值：标记为"待核实"，通知教师手动确认 |
|  | 5b. 学生未注册人脸：标记为"未知人员" |
| **后置条件** | 本次考勤记录已持久化，缺勤学生触发异常提醒判断 |

## 8. 术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| 人脸检测 | Face Detection | 在图像中定位人脸位置，输出边界框（Bounding Box） |
| 人脸识别 | Face Recognition | 将人脸与已知身份对应，本系统采用特征匹配方式 |
| 特征向量 | Feature Vector | CNN 提取的高维数值数组，用于表达人脸的抽象特征 |
| 余弦相似度 | Cosine Similarity | 衡量两个向量夹角的指标，值越接近1表示越相似 |
| 迁移学习 | Transfer Learning | 将已在大数据集（ImageNet）上预训练的模型用于新任务的技术 |
| MTCNN | Multi-task CNN | 多任务级联卷积神经网络，用于人脸检测与关键点定位 |
| JWT | JSON Web Token | 无状态认证令牌，包含用户身份信息和过期时间的加密字符串 |
| 考勤任务 | Attendance Task | 一次完整的课堂考勤过程，包含开始时间、结束时间和考勤结果 |
| 待核实 | Unverified | 识别置信度介于低阈值和高阈值之间，需教师人工确认的状态 |
| 置信度 | Confidence | 识别结果的可靠性得分，本系统取余弦相似度值（0~1） |
| 特征数据库 | Feature Database | 存储所有已注册学生特征向量的结构，加载于内存中供快速匹配 |
| Blueprint | Blueprint | Flask 中组织路由的模块单元，等同于其他框架的 Controller |
