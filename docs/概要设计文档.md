## 1. 系统架构设计

### 1.1 总体架构

系统采用**前后端分离 + AI推理服务**的三层架构：

```
┌─────────────────────────────────────────────────────┐
│                   前端层（Browser）                  │
│           Vue.js 3 + Element Plus + ECharts          │
│   学生管理 / 考勤任务 / 报表统计 / 系统管理          │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP/WebSocket (JSON)
┌──────────────────────▼──────────────────────────────┐
│                后端应用层（Flask）                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐ │
│  │用户认证  │ │学生档案  │ │考勤任务  │ │报表   │  │
│  │Blueprint │ │Blueprint │ │Blueprint │ │Blueprint│ │
│  └──────────┘ └──────────┘ └──────────┘ └────────┘ │
│  ┌─────────────────────────────────────────────────┐│
│  │              AI推理模块（独立服务）              ││
│  │  OpenCV人脸检测 → CNN特征提取 → 余弦相似度匹配  ││
│  └─────────────────────────────────────────────────┘│
└──────────────────────┬──────────────────────────────┘
                       │ SQLAlchemy ORM
┌──────────────────────▼──────────────────────────────┐
│                  数据持久层                          │
│         MySQL 8.0          本地文件系统              │
│   (结构化业务数据)        (人脸图片/模型权重)        │
└─────────────────────────────────────────────────────┘
```

### 1.2 技术选型说明

| 层次 | 技术 | 选型理由 |
|------|------|----------|
| 前端框架 | Vue.js 3 + Vue Router + Pinia | 组件化开发，生态成熟，任务书指定 |
| UI 组件库 | Element Plus | 后台管理系统首选，表单/表格组件完善 |
| 数据可视化 | ECharts | 考勤报表图表展示 |
| 后端框架 | Flask 3.x | 轻量灵活，与 Python AI 生态无缝集成 |
| 数据库 | MySQL 8.0 | 任务书指定，关系型数据适合业务数据 |
| ORM | SQLAlchemy 2.x | 安全防注入，模型映射清晰 |
| 人脸检测 | OpenCV + MTCNN | 多级联检测，准确率高于 Haar Cascade |
| 特征提取 | VGG16 / ResNet50（PyTorch） | 任务书指定对比两种模型 |
| 认证 | JWT（Flask-JWT-Extended） | 无状态，前后端分离适配 |
| 跨域 | Flask-CORS | 开发阶段解决跨域问题 |
| 接口文档 | Flask-RESTx（Swagger） | 自动生成 API 文档 |

### 2.3 AI 推理模块详细职责

| 子模块 | 职责 | 核心技术 |
|--------|------|----------|
| `face_detector.py` | 输入图像帧，输出人脸区域 bounding box 列表 | MTCNN |

## 5. AI 模型设计

### 5.1 整体推理流程

```
输入图像帧
    │
    ▼
[MTCNN 人脸检测]
    │ 输出：人脸 bounding boxes + 关键点
    ▼
[OpenCV 预处理]
    ├── 裁剪人脸区域（加 20px 边距）
    ├── 缩放至 224×224
    └── 归一化（ImageNet 均值/标准差）
    │
    ▼
[CNN 特征提取]（VGG16 / ResNet50 去掉最后分类层）
    │ 输出：512/2048 维特征向量
    ▼
[L2 归一化]
    │
    ▼
[余弦相似度匹配]（与内存中所有学生特征向量对比）
    │ 找最近邻
    ▼
[阈值判断]
    ├── similarity ≥ 0.75 → 识别成功，返回 student_id
    ├── 0.5 ≤ similarity < 0.75 → 标记 unverified
    └── similarity < 0.5 → 未知人员
```

### 5.2 迁移学习方案

| 阶段 | 操作 | 说明 |
|------|------|------|
| 加载预训练权重 | 使用 ImageNet 预训练的 VGG16/ResNet50 | `torchvision.models` |
| 冻结底层 | 冻结前 N 个 Block | 保留通用特征，降低过拟合风险 |
| 替换分类头 | 去掉最后 FC 层，改为：FC(512) → BN → ReLU → FC(num_classes) | 适配人脸识别分类数 |
| 微调训练 | 解冻全部层，小学习率（1e-4）端到端训练 | |
| 特征提取模式 | 推理时去掉分类头，输出倒数第二层特征 | 用于余弦相似度匹配 |

### 5.3 数据集方案

| 数据来源 | 用途 | 说明 |
|----------|------|------|
| LFW（Labeled Faces in the Wild） | 预训练/基准评估 | 13,000+ 张，1680+ 人 |
| MS-Celeb-1M（子集） | 增强泛化能力 | 选取 1000 人子集 |
| 自建校园数据集（脱敏） | Fine-tune 目标域 | 每人≥5张，不同角度/光线 |

## 6. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 识别方式 | 特征匹配（非分类） | 新增学生无需重训分类头，只需提取特征向量入库 |
| 人脸检测 | MTCNN | 多尺度检测，比 Haar Cascade 准确率高 ~15% |
| 特征维度 | 512d（VGG16）/ 归一化后 512d（ResNet50 avg pool） | 平衡精度与匹配速度 |
| 特征存储 | numpy `.npy` 文件 + 内存加载 | 百人级别匹配 <1ms，无需向量数据库 |
| 实时流处理 | WebSocket 推送识别结果 | 避免前端轮询，延迟更低 |
| 模型对比评估指标 | Top-1 准确率、F1 Score、ROC-AUC、推理时间 | 论文实验对比章节需要 |

## 9. WebSocket 通信协议设计

考勤任务的实时识别结果通过 WebSocket 推送，避免前端轮询。

### 9.1 连接建立

```
URL:  ws://<host>/ws/attendance/<task_id>
Headers:
```

### 9.3 帧率控制策略

- 前端默认每 **500ms** 推送一帧（2 fps）
- 若上一帧尚未处理完（后端繁忙），前端自动跳过本帧，避免积压
- 教室人数 ≤30 时，推理延迟目标 ≤200ms/帧

## 10. 错误处理策略

### 10.1 HTTP 错误码规范

| HTTP 状态码 | 业务场景 | 前端处理建议 |
|-------------|----------|-------------|
| 400 | 请求参数校验失败 | 表单校验提示，显示具体字段错误 |
| 401 | Token 缺失或过期 | 清除本地 Token，跳转登录页 |
| 403 | 权限不足 | 提示"无权限操作"，不跳转 |
| 404 | 资源不存在 | 提示"未找到相关数据" |
| 409 | 业务冲突（如学号重复） | 提示具体冲突原因 |
| 422 | 数据格式正确但业务校验失败 | 显示后端返回的 message 字段 |
| 500 | 服务器内部错误 | 提示"服务异常，请联系管理员"，上报错误码 |

### 10.2 AI 模块异常处理

| 异常场景 | 处理方式 | 降级策略 |
|----------|----------|----------|
| 模型文件缺失 | 启动时检测，缺失则记录警告日志 | 返回 503，提示"模型未就绪" |
| 推理超时（>5s） | 超时中断，返回空结果 | 标记本帧为 unverified |
| 显存不足（GPU OOM） | 捕获 CUDA 异常，自动切换 CPU 模式 | CPU 推理，性能下降但不中断 |
| 人脸检测返回空 | 正常返回空列表 | 前端提示"当前帧未检测到人脸" |
| 特征库为空 | 考勤开始前校验 | 阻止任务启动，提示"该课程暂无已注册学生" |
